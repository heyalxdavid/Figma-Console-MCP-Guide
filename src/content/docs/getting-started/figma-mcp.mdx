The Figma MCP (Model Context Protocol) server gives your AI assistant direct access to your Figma designs. It can inspect layers, extract design tokens, read component properties, and understand your design files without you needing to describe them manually. Below we'll walk through how to set it up with your preferred AI tool.

## Step 1: Enable the MCP Server in Figma

Before connecting any AI tool, you need to turn on the MCP server in the Figma desktop app.

<Steps>

1. **Open the Figma desktop app**

   Make sure you have the [Figma desktop app](https://www.figma.com/downloads/) installed and updated to the latest version.

2. **Open a design file and switch to Dev Mode**

   Create or open a Figma Design file, then toggle to **Dev Mode** using the switch in the toolbar at the bottom (or press <kbd>Shift</kbd> + <kbd>D</kbd>).

3. **Enable the desktop MCP server**

   In the **MCP server** section of the inspect panel, click **Enable desktop MCP server**. You should see a confirmation message that the server is running.

   ![Figma Dev Mode MCP server settings showing Enable Code Connect and Disable MCP server options](/Figma-Console-MCP-Guide/images/figma-mcp/figma-dev-mode-mcp.png)

</Steps>

<Aside type="tip">
The desktop MCP server runs locally at `http://127.0.0.1:3845/mcp`.
</Aside>

## Step 2: Connect Your AI Tool

<Tabs>
<TabItem label="Cursor">

### Let AI set it up for you

Open a new chat in Cursor (Agent mode) and paste this prompt:

> Add the Figma desktop MCP server to my mcp.json. The server URL is `http://127.0.0.1:3845/mcp` and the server name should be `figma-desktop`. Here are the Figma docs for reference: https://developers.figma.com/docs/figma-mcp-server/local-server-installation/

The agent will edit your `mcp.json` for you. Once it's done, verify the connection in your MCP settings (see step 3 below).

### Or set it up manually

<Steps>

1. **Open the MCP settings**

   Open the command palette with <kbd>Cmd</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> and search for **MCP**.

   ![Command palette showing MCP search results](/Figma-Console-MCP-Guide/images/figma-mcp/command-palette-mcp.png)

   Select **Cursor Settings: Tools & MCPs**.

   ![Cursor Settings sidebar with Tools & MCPs selected](/Figma-Console-MCP-Guide/images/figma-mcp/cursor-tools-mcps.png)

   You can also get there from the menu bar: **Cursor > Settings... > Cursor Settings**, then select **Tools & MCPs** in the sidebar.

   ![Cursor menu showing Settings path](/Figma-Console-MCP-Guide/images/figma-mcp/cursor-settings-menu.png)

2. **Add a new MCP server**

   Click **New MCP Server**.

   ![New MCP Server button](/Figma-Console-MCP-Guide/images/figma-mcp/new-mcp-server.png)

   This opens your `mcp.json` file. Add the Figma desktop server configuration:

   ```json
   {
     "mcpServers": {
       "figma-desktop": {
         "url": "http://127.0.0.1:3845/mcp"
       }
     }
   }
   ```

   Your `mcp.json` may already have other MCP servers listed. Just add the `"figma-desktop"` entry inside the existing `"mcpServers"` object.

3. **Verify the connection**

   After saving, you should see the Figma MCP server listed with a green indicator in your MCP settings.

   ![Figma MCP connected with green indicator](/Figma-Console-MCP-Guide/images/figma-mcp/figma-mcp-connected.png)

</Steps>

</TabItem>

### Or set it up manually

Claude Desktop has two places that relate to Figma. Make sure you use the right one:

- **Connectors** is for the remote Figma MCP server (connects directly to Figma's cloud). You don't need the Figma desktop app for this.
- **Developer** is where you add local MCP servers, including the Figma desktop MCP server.

![Claude Desktop Connectors page showing Figma as an Interactive connector](/Figma-Console-MCP-Guide/images/figma-mcp/claude-desktop-connectors.png)

![Claude Desktop Developer page showing Local MCP servers](/Figma-Console-MCP-Guide/images/figma-mcp/claude-desktop-developer.png)

To connect the **desktop** MCP server:

<Steps>

1. **Open Settings**

   In Claude Desktop, go to **Settings > Developer > Edit Config**.

2. **Add the Figma MCP server**

   Add the following to your `claude_desktop_config.json`:

   ```json
   {
     "mcpServers": {
       "figma-desktop": {
         "url": "http://127.0.0.1:3845/mcp"
       }
     }
   }
   ```

3. **Restart Claude Desktop**

   Restart Claude Desktop for the new server to take effect. You should see the Figma server listed as **running** in **Settings > Developer**.

</Steps>

## Using It

Once connected, you can provide Figma context to your AI assistant in two ways:

- **Selection-based:** Select a frame or layer in the Figma desktop app, then ask your AI tool to implement your current selection.
- **Link-based:** Copy the link to a frame or layer in Figma and paste it into the chat. The AI will extract the node ID to fetch the right design data.

### Try it out

A quick way to verify everything is working: open any Figma file, right-click a frame, and copy its link (**Copy/Paste as > Copy link**). Paste the link into your AI tool and ask:

> Can you see this design? Describe what you see.

If the MCP is connected correctly, your AI tool will describe the frame's contents, layout, and components.

### More example prompts

- "Implement the design I have selected in Figma"
- "What components are in this Figma file?"
- "Extract the color palette from this design"
- "Describe the layout of this frame"

## Troubleshooting

- **Connection failed:** Make sure the Figma desktop app is open and the MCP server is enabled in Dev Mode. The server must be running for your AI tool to connect.
- **No tools showing:** If your AI tool doesn't list any Figma tools, restart both the Figma desktop app and your AI tool.
- **Permission errors:** Ensure your Figma account has access to the files you're trying to read.
